# App Configuration for the Fast Android SDK

<embed src="/reusables/for-developers/_platform_all_sign_up_as_a_seller_banner_sandbox_and_contact_support.md" />

Configure the FastCheckout iOS SDK in `AndroidManifest.xml` with your **Fast App ID** :

1. Pass `appId` with the `meta-data` tag in `app/src/main/AndroidManifest.xml`

```kotlin
<application
	android:name="Your application name">

	<meta-data android:name="co.fast.app_id" android:value="${appId}" />

	<activity />
</application>
```

:::info App Startup library dependency

Android Jetpack's [App Startup](https://developer.android.com/topic/libraries/app-startup) library is used to initialize the Fast Android SDK.

:::

1. Enable `databinding` in your `app/build.gradle` . We use [databinding](https://developer.android.com/jetpack/androidx/releases/databinding) to bind views in the SDK and it is required for the `app` module to enable databinding.

```kotlin
android {

	compileSDKVersion ...
	//...

	buildFeatures {
    dataBinding true
  }
}
```

1. Add `jitpack` to your list of repositories in `root/build.gradle`. We use [Persistent Cookie Jar](https://github.com/thomas-bouvier/persistent-cookie-jar-okhttp) library for OkHttp to persist session data in our networking stack.

```kotlin
allprojects {
    repositories {
        ...
        maven { url "https://jitpack.io" }
    }
}
```

1. Initialize the `FastCheckout` class

```kotlin
private lateinit var fastCheckout: FastCheckout

override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        fastCheckout = FastCheckout(this, ::onFastCheckoutEvent)
}

// ::onFastCheckoutEvent explained in callbacks section
```

:::info Callbacks
You can find more details about `::onFastCheckoutEvent` in the [callbacks section of the Handle Fast Events page](../handle-fast-events#callbacks)
:::

## Add the Fast Checkout Button

On pages where you want the **FastCheckoutButton** to appear, generate the button and add it to your layout. You can integrate the Fast Checkout button with Android apps that use both XML-based and compose-based layouts.

1. Add the `FastCheckoutButton` to your XML or Compose layout

   - XML layout

     ```kotlin
     // xml layout

     <co.fast.android.sdk.checkout.FastCheckoutButton
       android:id="@+id/fast_checkout"
       android:layout_width="match_parent"
       android:layout_height="56dp" // we recommend 56dp
      />
     ```

   - Compose layout

     ```kotlin
     // jetpack compose layout

     FastCheckoutButton(
       onClick = { Fast.checkout(...) }, // launches checkout flow, explained below
       enabled = Boolean, // true if all variants selected, false otherwise
       modifier = Modifier
     	  .fillMaxWidth()
     )
     ```

<Alert variant="attention" header="Define invokeFastCheckout">
  The Fast Checkout button will not have an action installed by default. Instead
  you should implement `Fast.checkout` to call one of Fast's checkout functions.
</Alert>

More details about invoking Fast Checkout can be found [here](../use-fast-checkout)
